How To Deploy the Angular 19 Application to GitHub Pages Using angular-cli-ghpages

Introduction

Angular is a powerful framework for building modern web applications. GitHub Pages provides a free and easy way to host static websites directly from a GitHub repository. In this guide, we will walk through the steps to deploy an Angular 19 application to GitHub Pages using the angular-cli-ghpages tool. By following these instructions, your app will be published to https://<your-username>.github.io/<your-repository>.

Before you begin, make sure you have the following prerequisites in place:

  • Node.js and npm installed
  • An Angular 19 project (created with ng new)
  • Angular CLI installed globally (npm install -g @angular/cli)
  • Git installed and a GitHub account with a repository ready

Step-by-Step Guide

Step 1: Set Up Your Angular Project and initialize the git repo.

If you haven't already created your Angular project, start by running:

ng new my-angular-app

Navigate into the project folder and initialize a Git repository:

cd my-angular-app

Initialize a Git repository:

git init

Step 2: Add remote origin to your local git repo.

The git remote add origin repository-url command links your local Git project to a remote repository.

 git remote add origin https://github.com/github-username/remote-repo-name.git
  • origin is the default name for the remote.
  • remote-repo-name is the link to your GitHub repo.

Step 3: Add your changes to the git repo and commit the changes with small message.

git add .
git commit -m "Initial Commit"

Step 4: Set current branch to main and push the changes to remote repo.

git branch -M main

Set you current branch to main. In other words, the below command will renames your current branch to main

  • -M stands for “move” (force rename, even if main already exists).
  • It's useful if your branch is named master by default and you want to switch to the modern naming convention main.

git push -u origin main

Now the next step is to push your local main branch changes to remote repository.

  • -u sets the upstream, so future pushes can just use git push. (Once we set the upstream branch, then in further git push we can only use git push).

Step 5: Create the application Build.

Upon pushing the changes to the GitHub now it’s time to build the project. Building an Angular application means converting all your TypeScript, HTML, SCSS, and Angular code into pure, optimized HTML, CSS, and JavaScript files that browsers can actually understand and run. Building an Angular project prepares your app for production by making it small, fast, and ready for real users and browsers.

  • Compiles your TypeScript into JavaScript.
  • Optimizes your CSS and HTML.
  • Removes unused code (tree-shaking).
  • Compresses files (minification) for faster loading.
  • Creates a final version of your app inside the dist/ folder.
ng build --configuration production --base-href “https://github-username.github.io/remote-respository-name/”
Before Build: We have a project with lots of separate files (components, services, modules, etc.) written in Angular style (TS, HTML, CSS).
After Build: We get a few highly optimized .html, .css, and .js files inside /dist/your-app-name/ — ready to be deployed on a server like GitHub Pages, AWS, Netlify, etc.

Prerendered 1 static route.
Output location: F:\Upswing Assessment\Assignment\dist\assignment
NOTE: After the output folder is created, open it in File Explorer. Then, copy all files from the browser folder and any other folders, and paste them directly into the dist folder.

Step 6: Install the angular-cli-ghpages npm package.

The next step is to Install the angular-cli-ghpages npm package, The angular-cli-ghpages tool helps you easily deploy your Angular application to GitHub Pages with just a single command.

npm install -g angular-cli-ghpages

Step 7: Deploy the angular application.

The final step is to deploy your angular application on github.

npx angular-cli-ghpages –dir=dist/build-output-folder/github.com/github-username/remote-repo-name.git

build-output-folder: This will you get when you build the project, it will display the output folder path.

Step 8: Result

The final step is to deploy your angular application on github.

npx angular-cli-ghpages –dir=dist/build-output-folder/github.com/github-username/remote-repo-name.git

build-output-folder: This will you get when you build the project, it will display the output folder path.

Wait for the deployment to complete. You should see a message like “Successfully published!” in the console when done.

Step 9: Verify the Deployment

After deployment, open your browser and navigate to:

https://your-username.github.io/your-repo/

You should see your Angular app live on GitHub Pages. If you make updates to your app in the future, just rebuild and redeploy to publish the latest version.

Conclusion

Deploying an Angular 19 application to GitHub Pages is straightforward with the angular-cli-ghpages tool. By following this guide, you can quickly share your project online without needing a dedicated server. GitHub Pages handles the hosting for free, making it an ideal solution for front-end projects. Happy coding!
